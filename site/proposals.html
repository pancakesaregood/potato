How the proposal process works

1. Draft

Copy the proposal template. Fill in all eight required sections. Clear writing matters — incomplete sections are flagged automatically.

2. Open a PR

Submit your proposal file as a pull request. The CI pipeline runs immediately: markdown lint, template validation, rights compliance check, policy consistency check.

3. Review

Contributors and Governance Reviewers comment. You can revise and push changes. There is no time limit. The deliberation stage is the most important part.

4. Decision

If consensus is reached and all checks pass, a Maintainer merges the proposal. If not, it is archived with a record of why. Archived proposals can be reopened.

Submit a proposal

Proposals are submitted as GitHub Issues using a structured template. You will need a GitHub account. The template guides you through all eight required sections — you do not need to be an expert on any of them.
