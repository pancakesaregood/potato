# Distributed Ledger

## Summary

This addendum explores when a distributed ledger is useful for audit trails, public records, and delegated trust.

## Design Principles

- A ledger should be used only where it solves a real trust problem.
- Energy, complexity, and privacy costs must be justified.
- Off-chain and simpler options should always be considered first.

## Technical Mechanisms

- Use a ledger for tamper-evident logging of key governance events when appropriate.
- Store only minimal public proofs on shared infrastructure.
- Keep sensitive personal data off-chain and under conventional access controls.

## Validation Targets

- Ledger use cases are explicitly justified before adoption.
- Public proofs are verifiable without exposing personal records.
- Energy and operating overhead remain within published limits.
